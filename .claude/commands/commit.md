# Git 커밋 자동화

현재 변경사항을 분석하고 Conventional Commit 메시지와 Gitmoji로 커밋합니다.

## 실행 순서

1. `git status`로 변경사항 확인
2. `git diff`로 변경내용 상세 검토
3. 변경사항에 가장 적합한 커밋 타입과 Gitmoji 선택
4. `git add -A`로 모든 변경사항 스테이징
5. Conventional Commit 형식으로 커밋 메시지 작성 및 커밋

## Gitmoji 및 커밋 타입 가이드

### 기본 개발

- `🎉 Init`: 프로젝트 초기 세팅, 첫 커밋
- `✨ Feat`: 새로운 기능 추가
- `♻️ Refactor`: 코드 리팩터링 (기능 변화 없음)
- `🐛 Fix`: 버그 수정
- `🚑 HOTFIX`: 긴급한 버그 수정 (hotfix)
- `🔥 Remove`: 불필요한 코드/파일 제거

### 문서 및 타이포

- `📝 Docs`: 문서 수정 (README, 주석 등)
- `✏️ Typo`: 오타 수정

### 의존성 및 리소스

- `➕ Chore`: 라이브러리/패키지 추가
- `➖ Chore`: 라이브러리/패키지 제거
- `🍱 Chore`: 이미지/리소스 파일 추가 또는 수정

### 배포 및 인프라

- `✅ Test`: 테스트 코드 추가, 업데이트, 통과
- `🚀 Deploy`: 프로덕션 배포
- `🔧 Ci`: CI/CD 설정 변경 (GitHub Actions, Jenkins 등)
- `📦 Build`: 빌드 시스템, 환경설정 수정 (Vite, Webpack, Babel 등)
- `🔒 Fix`: 보안 관련 패치 (패키지 업데이트, 취약점 해결 등)

### 프론트엔드 전용

- `🎨 Style`: UI/스타일 변경 (CSS, 디자인 시스템)
- `🚸 Feat`: UX 개선
- `⚡ Perf`: 성능 개선 (렌더링 최적화, 코드 스플리팅, 레이지 로딩 등)
- `♿️ Feat`: 접근성 개선 (ARIA, 키보드 네비게이션 등)
- `💄 Style`: 마크업, 스타일 코드 포맷팅

### 백엔드 관련 (필요시)

- `🗃️ Feat`: 데이터베이스 스키마, 마이그레이션, 쿼리 변경
- `🔐 Feat`: 인증/인가 로직 추가 및 수정
- `🌐 Feat`: API 엔드포인트 추가/변경
- `🛠️ Chore`: 서버 설정, 미들웨어, 인프라 코드 변경

## 커밋 메시지 형식

```
<gitmoji> <type>: <subject>

[optional body]

[optional footer]
```

**예시**:

```
✨ feat: 사용자 프로필 편집 기능 추가

- 프로필 이미지 업로드 기능 구현
- 닉네임 중복 검사 API 연동
```

## 주의사항

- **Co-author 제외**: `Generated by claude-code` 또는 `Co-Authored-By: Claude <noreply@anthropic.com>` 등의 내용을 커밋 메시지에 포함하지 않습니다
- **한글 메시지**: 커밋 메시지는 한글로 명확하게 작성합니다
- **명확한 제목**: 커밋 제목은 50자 이내로 작성하고, 변경사항을 명확히 표현합니다
- **Body는 선택사항**: 복잡한 변경사항의 경우에만 본문을 추가합니다.
